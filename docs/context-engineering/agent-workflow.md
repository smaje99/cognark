# Agent Workflow

## Starting Work

1. Read `AGENT.md`.
2. Identify the active request.
3. Inspect the relevant architecture or context-engineering note.
4. If implementing backlog work, locate the next eligible requirement in the prioritized backlog.

## During Work

- Keep changes scoped.
- Use existing module boundaries.
- Add or update tests according to risk.
- Record material decisions in the relevant docs or ADRs.

## Finishing Work

- Run applicable checks.
- Report changed files and verification results.
- Update backlog status when the task is tied to a requirement.
- Note any blocked follow-up clearly.
